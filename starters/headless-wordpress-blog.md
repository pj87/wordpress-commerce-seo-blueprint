# Headless WordPress Blog Starter

## Goal

Use WordPress as a CMS and build a separate frontend for the public blog.

## Suggested Stack

- WordPress
- WordPress REST API
- Next.js
- TypeScript
- React

## WordPress Responsibilities

- Manage posts
- Manage pages
- Manage categories
- Manage authors
- Store media
- Provide REST API data

## Frontend Responsibilities

- Blog index
- Post detail pages
- Page rendering
- Category archives
- SEO metadata rendering
- Responsive UI
- Deployment

## Build Order

1. Set up WordPress and permalinks.
2. Create sample posts, pages, and categories.
3. Confirm REST API output.
4. Create frontend project.
5. Fetch posts and pages.
6. Render blog index and post detail.
7. Render categories.
8. Add metadata.
9. Deploy frontend.
10. Document API assumptions.

## Done Criteria

- Frontend renders posts from WordPress.
- Post detail pages work.
- Pages can be fetched and rendered.
- Categories work.
- Metadata is present.
- API setup is documented.
