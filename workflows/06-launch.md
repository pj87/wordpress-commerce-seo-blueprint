# Launch Workflow

## Goal

Move the WordPress site from build-ready to public-ready without missing critical setup.

## Inputs

- Completed site
- Domain access
- Hosting access
- Analytics access
- Search Console access
- Backup solution
- Email delivery method

## Steps

1. Connect domain.
2. Enable SSL.
3. Confirm homepage and blog page settings.
4. Configure permalinks.
5. Test navigation, footer, forms, search, and 404 page.
6. Configure SEO plugin, sitemap, titles, and descriptions.
7. Install analytics.
8. Connect Search Console.
9. Configure caching and image optimization.
10. Configure backups.
11. Review users and remove unused plugins.
12. Test mobile layouts.
13. Run checkout test if WooCommerce is active.
14. Submit sitemap.

## Outputs

- Public site
- Connected domain and SSL
- Working forms
- Analytics and Search Console
- Backup and security basics
- Launch notes

## Acceptance Criteria

- The public domain loads over HTTPS.
- Forms and checkout work.
- Sitemap is available.
- Analytics is installed.
- Backups are configured.
- No unused admin users or unnecessary plugins remain.
