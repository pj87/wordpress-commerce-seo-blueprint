# Maintenance Workflow

## Goal

Keep a WordPress site stable, secure, backed up, and commercially useful after launch.

## Weekly Steps

1. Check backups.
2. Check uptime.
3. Test important forms.
4. Review security alerts.
5. Check pending updates.

## Monthly Steps

1. Update plugins and themes carefully.
2. Update WordPress core when appropriate.
3. Test critical user flows.
4. Review analytics.
5. Review Search Console.
6. Optimize new images.
7. Clean spam comments.
8. Check broken links.

## Before Major Updates

1. Create a backup.
2. Test on staging if possible.
3. Update plugins.
4. Update theme.
5. Update WordPress core.
6. Test forms, checkout, navigation, and important pages.

## Outputs

- Maintenance log
- Update history
- Backup confirmation
- Issue list
- Improvement backlog

## Acceptance Criteria

- Backups are recent.
- Critical forms and checkout still work.
- Updates are applied deliberately.
- Security alerts are reviewed.
- Performance and SEO problems are tracked.
