# Landing Page Build Workflow

## Goal

Build a conversion-focused landing page that can be reused for service, product, or lead-generation projects.

## Inputs

- Offer
- Audience
- Primary CTA
- Secondary CTA if needed
- Brand assets
- Testimonials or trust signals
- Form requirements

## Steps

1. Build the hero section with headline, subheadline, CTA, and trust signal.
2. Add a problem section that reflects user pain.
3. Add a solution section that explains the offer.
4. Add benefits before feature details.
5. Add feature details only where they support the buying decision.
6. Add proof: testimonials, metrics, logos, case studies, reviews, or screenshots.
7. Add a simple process section.
8. Add FAQ items to handle objections.
9. Add a final CTA.
10. Test the form and mobile layout.
11. Add SEO title, meta description, clean slug, and image alt text.

## Outputs

- Completed landing page
- Working CTA
- Tested form
- SEO metadata
- Mobile-ready layout

## Acceptance Criteria

- The page explains who it is for and what action to take.
- CTA is visible near the top and repeated near the end.
- The form submits correctly.
- The page has one H1 and a logical heading structure.
- The page works on mobile.
