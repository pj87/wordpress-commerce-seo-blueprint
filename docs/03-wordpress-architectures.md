# WordPress Architectures

## Classic WordPress

Use a WordPress theme, plugins, the admin panel, PHP templates, and the block editor.

Best for:

- Fast landing pages
- Blogs
- Business sites
- WooCommerce stores
- Client websites

This is the recommended starting architecture.

## WordPress + WooCommerce

Use WordPress as the CMS and WooCommerce as the commerce layer.

Best for:

- Small online stores
- Digital product stores
- Local business stores
- Simple physical product catalogs

Focus first on standard WooCommerce flows before customizing checkout or account logic.

## Headless WordPress

Use WordPress as the CMS and build a separate frontend with a framework such as Next.js.

Best for:

- Custom frontend experiences
- React-based marketing sites
- Content APIs
- Cases where WordPress editing is needed but the frontend must be custom

Avoid headless WordPress early if the project can be delivered faster with classic WordPress.
