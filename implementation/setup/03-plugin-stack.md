# Plugin Stack

## Goal

Use a small, practical plugin stack for the demo site.

## Recommended Categories

- SEO
- Forms
- Cache/performance
- Backup
- Security
- Analytics

## Example Stack

Choose one plugin per category where possible:

| Category | Purpose |
| --- | --- |
| SEO | Titles, meta descriptions, sitemap, schema basics |
| Forms | Contact form and email notifications |
| Cache | Page cache and basic performance optimization |
| Backup | Scheduled database and file backups |
| Security | Login protection and basic hardening |
| Analytics | Tracking integration |

## Plugin Rules

- Install only what the project needs.
- Avoid overlapping plugins.
- Prefer established plugins with active maintenance.
- Remove unused plugins before launch.
- Test forms and checkout after plugin updates.

## Demo Requirement

The demo service site needs SEO, forms, cache, backup, security, and analytics coverage before launch.
